# 仿滴滴平台-前端

本仓库包含仿滴滴平台三端前端应用。后端仓库在同级目录 `../didi-taxi`，正常联调统一走后端网关 `http://127.0.0.1:18080`。

## 应用

| 应用 | 目录 | 默认端口 | 技术栈 | 说明 |
|---|---|---:|---|---|
| 乘客端 H5 | `didi-passenger-h5` | 5173 | Vue 3 + Vite + Vant | 登录、首页、下单/跟单、我的订单、设置、我的钱包、乘客 WS。 |
| 司机端 H5 | `didi-driver-h5` | 5174 | Vue 3 + Vite | 司机登录注册、听单、接拒单、行程推进、换队、司机 WS。 |
| 管理后台 | `didi-Vue` | 5175 | Vue 3 + Vite + Element Plus | 后台登录、动态菜单、订单管理、运力配置、计价管理、换队审核、系统用户。 |

## 启动

进入对应应用目录后执行：

```bash
npm install
npm run dev
npm run build
```

常用启动示例：

```bash
cd didi-passenger-h5
npm run dev
```

三端通过 `VITE_API_BASE_URL` 指向后端。当前开发环境默认走：

```text
http://127.0.0.1:18080
```

## 当前重点

- 乘客端个人中心已接入二期能力：我的订单、设置、我的钱包。
- 我的钱包入口顺序为：免密支付设置、银行卡、优惠券、借钱、车险。
- 我的钱包当前只实现免密支付设置和优惠券；银行卡、借钱、车险保留入口并提示待开发。
- 免密支付本期支持支付宝、微信；钱包摘要、协议列表、签约、默认设置、关闭均走 `/app/api/v1/wallet/**`。
- 优惠券列表走 `/app/api/v1/wallet/coupons`；优惠券规则和用券流水归后端 `calculate-service`，不是前端本地规则。
- 管理后台的车队营销优惠券页面仍处于讨论稿阶段，尚未开发。
